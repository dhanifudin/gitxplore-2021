```bash
man git

       git - the stupid content tracker

DESCRIPTION
       Git is a fast, scalable, distributed revision control system with an
       unusually rich command set that provides both high-level operations and
       full access to internals.

       See gittutorial(7) to get started, then see giteveryday(7) for a useful
```

<Steps>
  <Step><li>Open your terminal or git bash (in windows)</li></Step>
  <Step><li>Create a directory</li></Step>
  <Step><code className="language-bash">mkdir hello-git</code></Step>

# Check Repository Status

<Steps>
  <Step><li>Run following command, inside your repository</li></Step>
  <Step><code className="language-bash">git status</code></Step>
  <Step><li>The directory is called <b>Working Directory</b></li></Step>
  <Step><li>All changed files will be <b>tracked</b></li></Step>
  <Step><li>All new files are <b>untracked</b> by default</li></Step>
</Steps>

## Scenario 5: Resolve Conflict

![Git conflict](https://source.unsplash.com/random/800x600/?conflict)

---

```bash
$ git status
> # On branch master
> # You have unmerged paths.
> #   (fix conflicts and run "git commit")
> #
> # Unmerged paths:
> #   (use "git add ..." to mark resolution)
> #
> # both modified:      README.md
> #
> no changes added to commit (use "git add" and/or "git commit -a")
```
